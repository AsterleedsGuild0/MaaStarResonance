# 连接程序与确认实例

## 这篇文档解决什么问题

这篇文档负责帮助你启动程序、连接到正确的模拟器实例，并在多开场景下确认没有连错账号。

## 启动程序并连接模拟器

1. 运行 `MFAAvalonia.exe`。
2. 如果只开了一个模拟器，程序通常会自动连接。
3. 如果开了多个模拟器，请在下拉列表中选择目标实例。
4. 切换到左侧的 `截图` 页面，确认当前连接的是目标账号。

## 列表没有完整刷新时怎么办

如果列表里没有显示完整的模拟器信息，可以先切换到 `桌面应用`，等信息刷新后再切回模拟器页面。

![桌面应用刷新入口](../用户文档/新手上路/新手上路.assets/202510251044974.png)

![刷新后的模拟器信息](../用户文档/新手上路/新手上路.assets/202510251045001.png)

## 多开时如何确认 adb 端口

1. 多开场景下，需要根据 adb 端口选择正确实例。
2. 以 MuMu 为例，可以在 `设备诊断` 中查看当前模拟器端口号。
3. 将诊断页显示的端口与程序中的实例信息对应起来。
4. 再次切回 `截图` 页面，确认当前显示的是目标账号。

![adb 端口示意](../用户文档/新手上路/新手上路.assets/202510251049031.png)
