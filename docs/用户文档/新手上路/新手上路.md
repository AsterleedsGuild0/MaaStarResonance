# 新手上路

## 这篇文档适合谁

这篇文档适合第一次使用 MaaStarResonance 的用户，目标是完成环境准备、连接模拟器、运行第一组任务。

## 使用前提

- 当前主要支持 Windows。
- 当前推荐使用 MuMu 模拟器。
- 模拟器分辨率需为横屏 `16:9`，推荐 `1280x720`。

## 完成后你会得到什么

- 能正常启动程序。
- 能连接到目标模拟器。
- 能执行至少一组基础任务。

## 第一步：准备环境

1. 从 Release 页面下载并解压 `MaaStarResonance-win-x86_64.zip` 或 `MaaStarResonance-win-aarch64.zip`。
2. 在解压后的目录中找到 `DependencySetup_依赖库安装_win.bat`。
3. 双击运行该脚本，等待缺失依赖自动安装完成。

## 第二步：配置模拟器

1. 将 MuMu 模拟器设置为横屏 `16:9`。
2. 将分辨率设置为 `1280x720`。
3. 如果你的模拟器没有默认的 `1280 x 720 (DPI 240)` 选项，请手动创建对应配置。
4. 推荐使用独显渲染；如果你使用 AMD CPU + AMD 显卡且帧率异常低，再根据需要调整显卡面板设置。

![MuMu 分辨率入口](./新手上路.assets/image-20251102164845880.png)

如果没有现成的 `1280 x 720 (DPI 240)` 选项，可以参考下图填写自定义分辨率：

![MuMu 预设分辨率](./新手上路.assets/image-20251102165004973.png)

![MuMu 自定义分辨率](./新手上路.assets/image-20251102165111692.png)

## 第三步：确认游戏图像设置

1. 启动游戏并进入常用界面。
2. 尽量使用项目文档推荐的图像质量配置。
3. 如果你使用与推荐配置差异较大的画质，后续识别效果可能不稳定。

![推荐图像设置](./新手上路.assets/image-20251102165400165.png)

## 第四步：启动程序并连接模拟器

1. 运行 `MFAAvalonia.exe`。
2. 如果只开了一个模拟器，程序通常会自动连接。
3. 如果开了多个模拟器，请在下拉列表中选择目标实例。
4. 切换到左侧的 `截图` 页面，确认当前连接的是目标账号。

![程序启动界面](./新手上路.assets/202510251041408.png)

![模拟器连接入口](./新手上路.assets/202510251042894.png)

![多开时选择目标实例](./新手上路.assets/202510251046665.png)

如果列表里没有显示完整的模拟器信息，可以先切换到 `桌面应用`，等信息刷新后再切回模拟器页面。

![桌面应用刷新入口](./新手上路.assets/202510251044974.png)

![刷新后的模拟器信息](./新手上路.assets/202510251045001.png)

## 第五步：确认 adb 端口

1. 多开场景下，需要根据 adb 端口选择正确实例。
2. 以 MuMu 为例，可以在 `设备诊断` 中查看当前模拟器端口号。
3. 将诊断页显示的端口与程序中的实例信息对应起来。

![设备诊断入口](./新手上路.assets/202510251047770.png)

![adb 端口示意](./新手上路.assets/202510251049031.png)

## 第六步：开始任务

1. 等待程序完成 Python agent 启动和依赖检查。
2. 回到主页，勾选要执行的任务。
3. 点击 `开始任务`，确认任务开始运行。

![初始化过程](./新手上路.assets/202510251050026.png)

![截图页确认账号](./新手上路.assets/202510251053587.png)

![主页开始任务](./新手上路.assets/202510251054777.png)

## 使用建议

1. 可以在左侧任务列表中勾选多个任务，程序会按顺序执行。
2. 如果你需要重复执行某个任务，可以复制该任务后组合使用。
3. 先完成简单日常再执行耗时任务，通常更容易观察配置是否正确。

## 限制与注意事项

- 不满足推荐分辨率时，运行异常不在支持范围内。
- 多开时需要根据 adb 端口选择正确实例。
- 部分图像质量配置未完整测试覆盖。
- 当前文档主要基于 Windows + MuMu 模拟器环境编写。
