# 文档编写规范

## Docusaurus Markdown 格式规范

### Front Matter 元数据

每个文档文件顶部应包含 YAML front matter，用于配置文档的元数据：

```markdown
---
id: doc-id                          # 文档唯一标识符（可选，默认为文件名）
title: 文档标题                      # 页面标题
sidebar_label: 侧边栏显示名称         # 侧边栏中显示的文本（可选）
sidebar_position: 1                 # 侧边栏中的排序位置（数字越小越靠前）
description: 文档描述                # SEO 描述
keywords:                           # SEO 关键词
  - 关键词1
  - 关键词2
tags: [标签1, 标签2]                # 文档标签
hide_title: false                   # 是否隐藏标题
hide_table_of_contents: false       # 是否隐藏目录
---
```

### 常用 Front Matter 字段说明

| 字段 | 说明 | 示例 |
| ------ | ------ | ------ |
| `sidebar_position` | 控制文档在侧边栏中的顺序 | `sidebar_position: 2` |
| `sidebar_label` | 自定义侧边栏显示文本 | `sidebar_label: '快速开始'` |
| `title` | 页面标题（会显示在浏览器标签） | `title: 新手指南` |
| `description` | 页面描述（用于 SEO） | `description: 如何快速上手` |
| `tags` | 文档分类标签 | `tags: [教程, 入门]` |
| `draft` | 标记为草稿（开发环境可见） | `draft: true` |
| `unlisted` | 不在侧边栏显示但可访问 | `unlisted: true` |

### Markdown 基础语法

```markdown
# 一级标题
## 二级标题
### 三级标题

**粗体文本**
*斜体文本*
~~删除线~~

[链接文本](https://example.com)
[内部链接](./other-doc.md)

![图片描述](/img/image.png)
```

### 代码块

使用三个反引号包裹代码，并指定语言以启用语法高亮：

````markdown
```javascript
function hello() {
  console.log('Hello, world!');
}
```

```python
def hello():
    print("Hello, world!")
```
````

支持的特性：

- 添加标题：````markdown ```js title="example.js"````
- 高亮行：````markdown ```js {1,3-5}````
- 显示行号：````markdown ```js showLineNumbers````

### 告示框（Admonitions）

用于突出显示重要信息：

```markdown
:::note
这是一个普通提示。
:::

:::tip
这是一个小技巧。
:::

:::info
这是一条信息。
:::

:::warning
这是一个警告。
:::

:::danger
这是一个危险提示。
:::
```

自定义标题：

```markdown
:::tip[自定义标题]
这里是内容。
:::
```

### 选项卡（Tabs）

用于展示多种选择或多语言代码示例：

````mdx
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';

<Tabs>
  <TabItem value="js" label="JavaScript">
    ```js
    console.log('Hello');
    ```
  </TabItem>
  <TabItem value="py" label="Python">
    ```python
    print('Hello')
    ```
  </TabItem>
</Tabs>
````

## 通用写法

- 开头先写"这篇文档适合谁"。
- 前 20 行给出结论、限制、前置条件。
- 一个页面只解决一个核心问题。
- 操作步骤使用有序列表。
- 注意事项使用单独的"限制 / 风险 / 常见问题"标题。
- 截图只保留能帮助决策或定位控件的图片。

## 模板一：入口页

1. 这是什么。
2. 我该先看哪一篇。
3. 按场景查找。
4. 注意事项。

## 模板二：功能使用页

1. 功能简介。
2. 使用前提。
3. 配置步骤。
4. 特殊限制。
5. 常见问题。

## 模板三：开发者说明页

1. 适用对象。
2. 前置环境。
3. 操作步骤。
4. 验证方式。
5. 参考资料。

## 术语约定

- "入口页"负责分流和导航，不负责写完整操作细节。
- "功能使用页"只解释一个功能或一组紧密相关的操作。
- "开发者说明页"默认面向已经准备进入仓库修改内容的贡献者。

## 图片使用建议

- 一组连续步骤优先保留 1 到 3 张关键截图。
- 截图文件名尽量与场景对应，避免难以维护的重复图片。
- 如果图片不能帮助用户判断按钮位置、状态差异或配置结果，就不要保留。
