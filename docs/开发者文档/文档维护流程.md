# 文档维护流程

## 什么时候必须更新文档

- 新增用户可见功能。
- 修改现有任务流的前置条件、限制或配置方式。
- 修改开发环境、构建方式或提交流程。

## 修改文档时的最小检查清单

1. 是否更新了对应入口页。
2. 是否更新了对应功能页或开发页。
3. 是否运行 `npx markdownlint-cli2` 检查 Markdown。
4. 是否运行 `npm run docs:check`。
5. 是否确认 README 中没有过期重复描述。

## 提交建议

- 小步提交，一个主题一个 commit。
- 文档结构调整与内容重写分开提交。
- 如果修改了入口页，同时附上站点截图或链接供评审快速确认。

## 推荐审阅顺序

1. 先检查入口页是否仍能把人带到正确页面。
2. 再检查具体功能页或开发页是否与当前实现一致。
3. 最后运行站点构建，确认没有 broken link 或 sidebar 错误。
